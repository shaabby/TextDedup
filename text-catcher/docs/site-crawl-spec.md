# text-catcher 项目需求说明（草案）

## 1. 项目目标

在 `text-catcher` 中实现一个面向目标网站的文本采集能力，满足以下核心要求：

1. 爬取目标网站的全站文本内容（在设定边界内递归遍历站内页面）。
2. 只抓取 HTML 页面中的文本，不抓取图片及其他二进制资源。

## 2. 核心需求

### 2.1 全站文本爬取

- 输入：一个或多个目标域名（如 `www.example.com`）。
- 行为：
  - 从域名首页（`https://<domain>/`）开始抓取页面。
  - 解析页面中的站内链接并继续抓取（递归/BFS 均可）。
  - 对 URL 去重，避免重复抓取。
  - 支持最大深度、最大页面数、允许域名列表等边界控制，避免无限爬取。
- 输出：每个有效页面生成一条结构化文档记录（如 `doc_id/source/title/url/language/text`）。

### 2.2 仅抓 HTML 文本

- 仅请求并处理 `text/html` 内容。
- 只提取页面可读文本（标题、正文等），去除脚本和样式内容。
- 明确不抓取以下资源：
  - 图片（`jpg/png/webp/gif/svg` 等）
  - 视频、音频
  - 压缩包、Office/PDF 等附件
  - CSS、JS 等静态资源
- 非 HTML 响应应记录为“跳过”并计数，不进入正文存储。

## 3. 非功能需求

- 稳定性：支持超时重试、指数退避、限速（遵守站点访问频率）。
- 可恢复：支持断点续跑，避免重复抓取已完成 URL。
- 可观测：记录抓取成功数、失败数、跳过数（含非 HTML 跳过）。
- 可配置：抓取边界与并发参数通过 `config.yaml` 管理。

## 4. 建议配置项（新增）

建议在 `config.yaml` 中增加如下字段：

- `targets.domains`: 目标域名列表（核心入口配置）。
- `targets.default_language`: 默认语言标记。
- `crawl.recursive_enabled`: 是否开启全站递归抓取。
- `crawl.allowed_domains`: 允许抓取的域名列表。
- `crawl.max_depth`: 最大抓取深度。
- `crawl.max_pages`: 最大抓取页面数。
- `crawl.follow_query_strings`: 是否保留 query 参数参与 URL 去重。
- `crawl.only_html`: 固定为 `true`（只处理 HTML）。
- `crawl.skip_binary_resources`: 固定为 `true`（跳过图片等二进制资源）。

## 5. 验收标准

满足以下条件则视为需求达成：

1. 对给定站点入口执行后，系统可自动扩展抓取到站内其他页面（非仅固定 URL 列表）。
2. 存储结果中仅包含 HTML 页面抽取文本，不包含图片或其他二进制资源内容。
3. 日志中可看到非 HTML 资源被识别并跳过的统计。
4. 第二次运行可复用状态并跳过已抓取 URL。

## 6. 非目标（当前阶段不做）

- 不做 JS 渲染页面抓取（如需浏览器渲染可后续扩展）。
- 不做 OCR、图片文本识别。
- 不做同义词替换、改写、回译、样本对生成。

## 7. 交付建议

按以下顺序实施：

1. 增加站内链接发现与 URL 队列机制（BFS）。
2. 增加 URL 规范化与去重策略。
3. 增加 HTML Content-Type 检查与二进制资源过滤。
4. 完成日志与状态落盘字段扩展。
5. 使用一个中小型站点做端到端验证。
